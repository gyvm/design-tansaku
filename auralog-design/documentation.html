Auralog Design Documentation

Complete guidelines for implementing the Auralog design system across Windows and macOS desktop applications.

Introduction

The Auralog design system is built to create a modern, calm, and intelligent user experience for automatic meeting transcription and AI-powered note generation. This documentation provides comprehensive guidelines for maintaining consistency across all touchpoints.

Design Philosophy

Auralog's design communicates trust, productivity, and precision through a semi-professional aesthetic similar to Notion, Linear, or Raycast. The design is minimal but not cold, with subtle gradients used sparingly for accent elements.

Design Principles

1. Clarity First

Every interface element should have a clear purpose. Avoid unnecessary decoration or complexity that doesn't serve the user's goal.

2. Calm Intelligence

The design should feel intelligent without being overwhelming. Use subtle AI indicators and smooth transitions to communicate processing without anxiety.

3. Consistency Across Platforms

While respecting platform conventions (Windows/macOS), maintain visual consistency in colors, typography, spacing, and component behavior.

4. Progressive Disclosure

Show information when needed. Complex features should be accessible but not immediately visible to avoid overwhelming new users.

Getting Started

To implement the Auralog design system in your project, include the design system CSS file:

<link rel="stylesheet" href="design-system.css">

This provides access to all CSS custom properties (variables) that define the design system:

/* Example usage */ .my-component { color: var(--color-primary); padding: var(--space-4); border-radius: var(--radius-md); transition: all var(--transition-fast); }

Dark Mode Considerations

All colors automatically adapt for dark mode using the data-theme="dark" attribute. Never hard-code light mode colors in your components.

Typography

Font Family

Auralog uses system fonts for optimal performance and native feel:

font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Inter', 'Roboto', 'Helvetica Neue', sans-serif;

Type Scale

Style Size Weight Line Height Use Case
Display 48px Bold (700) 1.2 Hero titles, landing pages
H1 36px Bold (700) 1.2 Page titles
H2 28px Semibold (600) 1.2 Section headers
H3 22px Semibold (600) 1.5 Subsection headers
H4 18px Medium (500) 1.5 Card titles, labels
Body 15px Regular (400) 1.5 Primary content
Body Small 14px Regular (400) 1.5 Secondary content
Caption 12px Regular (400) 1.5 Meta information, labels
Typography Best Practices
  • Maintain a clear hierarchy with consistent heading levels
  • Use sentence case for most UI text; avoid all caps except for small labels
  • Keep line lengths between 50-75 characters for optimal readability
  • Use medium or semibold weights for emphasis, not bold

Spacing Guidelines

  • --space-2 to --space-4: Internal component padding, gaps between related elements
  • --space-4 to --space-6: Default component padding, card spacing
  • --space-6 to --space-8: Section padding, content margins
  • --space-12 to --space-16: Large section spacing, page margins

Elevation & Shadows

Shadows create depth and hierarchy. Auralog uses four shadow levels:

Level Variable Use Case
Small --shadow-sm Subtle elevation for cards, buttons at rest
Medium --shadow-md Hover states, active cards
Large --shadow-lg Dropdowns, popovers
Extra Large --shadow-xl Modals, important overlays
Shadow Usage

Use shadows sparingly. Too many elevated elements create visual noise. Reserve xl shadows for modals and critical UI elements only.

Border Radius

Consistent border radius creates cohesive, modern interfaces:

Size Value Use Case
--radius-sm 4px Small UI elements, tags, badges
--radius-md 6px Buttons, inputs, small cards
--radius-lg 10px Cards, panels, content blocks
--radius-xl 14px Modals, large containers
--radius-full 9999px Pills, circular elements, avatars

Responsive Design

Desktop-First Approach

Auralog is designed for desktop applications with the following considerations:

  • Minimum Width: 1024px (narrow laptop display)
  • Optimal Width: 1440px (standard desktop)
  • Maximum Content Width: 1400px for readability
  • Ultra-Wide Support: Centered content with max-width constraints

Layout Patterns

Sidebar + Content

Used in Dashboard and Settings:

.layout { display: flex; .sidebar { width: 240px; } .content { flex: 1; max-width: 1400px; } }

Split Panel

Used in Transcript Workspace:

.workspace { display: flex; .transcript { flex: 1; } .summary { width: 400px; } }

Animation Guidelines

Animation Principles

  1. Purposeful: Every animation should serve a functional purpose
  2. Subtle: Animations should enhance, not distract
  3. Fast: Keep durations short (150-350ms)
  4. Natural: Use easing functions for realistic motion

Transition Speeds

Speed Duration Use Case
--transition-fast 150ms Hover effects, color changes, simple state transitions
--transition-normal 250ms Component entrance/exit, panel slides, opacity changes
--transition-slow 350ms Complex animations, large movements, modal appearances

Easing Function

All transitions use a consistent easing curve:

cubic-bezier(0.4, 0, 0.2, 1) /* Material Design standard easing */
Animation Best Practices
  • Use transform and opacity for smooth 60fps animations
  • Avoid animating width, height, or layout properties
  • Respect user's motion preferences (prefers-reduced-motion)
  • Keep loading animations looped and smooth

Accessibility

Color Contrast

All text and interactive elements meet WCAG 2.1 AA standards:

  • Primary text: 7:1 contrast ratio (AAA)
  • Secondary text: 4.5:1 contrast ratio (AA)
  • Interactive elements: 3:1 contrast ratio minimum

Keyboard Navigation

  • All interactive elements must be keyboard accessible
  • Tab order should follow logical visual flow
  • Focus states must be clearly visible
  • Provide keyboard shortcuts for common actions

Screen Readers

  • Use semantic HTML (button, nav, main, etc.)
  • Provide alt text for icons and images
  • Use ARIA labels where semantic HTML isn't sufficient
  • Announce dynamic content changes

Do's and Don'ts

Color Usage

Do
  • Use primary color for main CTAs
  • Maintain consistent semantic colors
  • Test all colors in both themes
  • Use neutral colors for backgrounds
Don't
  • Use multiple primary colors
  • Hard-code colors; use variables
  • Use red/green alone to convey info
  • Overwhelm with gradients

Spacing

Do
  • Use spacing tokens consistently
  • Maintain visual rhythm
  • Give elements room to breathe
  • Group related elements
Don't
  • Use arbitrary pixel values
  • Cram too much in one view
  • Vary spacing inconsistently
  • Ignore the 8px grid

Typography

Do
  • Maintain clear hierarchy
  • Use appropriate line heights
  • Left-align body text
  • Keep line lengths readable
Don't
  • Use more than 3 font weights
  • Center long paragraphs
  • Use all caps for long text
  • Mix too many font sizes
